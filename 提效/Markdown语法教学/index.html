Markdown 语法教程

Markdown 简介​

什么是 Markdown​

  • 一种轻量级的标记语言，用于简单、易读易写的文本格式转换
  • 最初由约翰·格鲁伯（John Gruber）和亚伦·斯沃茨（Aaron Swartz）开发
  • 广泛用于编写文档、发布文章、撰写博客和编写文本编辑器的格式化工具

Markdown 的设计目标​

  • 使人们可以专注于内容而不是样式
  • 提供了一组简单的标记语法，可以使用纯文本的形式来添加标题、段落、列表、链接、图片和其他常见的文本格式

Markdown 的特点​

  • 平台无关性，可以在几乎所有的文本编辑器和网页浏览器中使用
  • 文件扩展名通常是".md"或".markdown"
  • 人性化的书写模式
    • 自动排版
    • 自动目录
    • 保留格式易于转载
    • 易于编辑、发布、阅读
  • 广泛应用
    • 博客平台：简书、知乎、CSDN、Github 等
    • 笔记平台： 有道云笔记，印象笔记等
    • 本地平台：Typora、VS Code、MacDown、sublime Text + 插件等

编写软件​

一个墙裂推荐的软件是 Typora（付费）：

  • 官网：https://typora.io/
  • 官方中文站：https://typoraio.cn/

如果你不愿意付费也可以用别的，例如 VS Code，下载速度过慢的问题可以参考 这里

如何选择？

Typora 和 VS Code 都是我日常在使用的，但是 Typora 更常用，本文档也是用它编写的

各个软件的差别其实并不大，Typora 的主要优势在于它的实时预览、所见即所得的编辑方式和美观的界面

Markdown 基础语法规范​

开始之前
  • 如果你使用的是 Typora，可以使用快捷键 Ctrl+/ 切换到源代码格式查看源代码
  • 如果你使用的是 VS Code，安装 Markdown Preview Enhanced 这个插件，预览 Markdown 文件的快捷键为 Ctrl+K V
  • 以下所有语法的符号均为英文字符
  • 提到的快捷键均针对 Windows 键盘
  • 在书写 markdown 时，段落之间、引用块之间、代码块之间和不同类型的语法之间通常需要空一行，否则渲染出来的格式可能会与预期不符，但在大多数时候无需担心这个问题

标题​

# + 一个空格 + 内容

# 的数量代表标题的级别

例如：

# 我是一级标题

## 我是二级标题

### 我是三级标题

的渲染结果如下：

http://localhost:3000

我是一级标题

我是二级标题​

我是三级标题​

斜体、加粗、加粗斜体、删除线​

在文本两边加上不同数量的 * 或者 _，即可分别实现前三种效果，删除线则是两对 ~，例如：

这是*斜体*

这也是_斜体_

这是**加粗**

这也是__加粗__

这是***斜体且加粗***

这也是___斜体且加粗___

~~这不是删除线~~

的渲染结果如下：

http://localhost:3000

这是斜体

这也是斜体

这是加粗

这也是加粗

这是斜体且加粗

这也是斜体且加粗

这不是删除线

为了看起来美观我更喜欢用 *，但实际上在 Typora 和 VS Code 中还有更快捷的方法来加粗（Ctrl+B）加斜体（Ctrl+I）

有序列表​

语法为：数字 + . + 一个空格 + 内容

例如：

1. 这是第一行
2. 这是第二行
    1. 这是嵌套第一行
    2. 这是嵌套第二行

的渲染结果如下：

http://localhost:3000
  1. 这是第一行
  2. 这是第二行
    1. 这是嵌套第一行
    2. 这是嵌套第二行

无序列表​

语法与有序列表类似：只需要把数字换为 * 或者 + 或者 -

例如：

- 这是无序列表的一项

* 这是无序列表的一项

- 这是无序列表的一项
    - 这是嵌套无序列表的一项
    * 这是嵌套无序列表的一项
    - 这是嵌套无序列表的一项

可见同级的项目不管使用哪个符号的渲染效果都是相同的：

http://localhost:3000
  • 这是无序列表的一项
  • 这是无序列表的一项
  • 这是无序列表的一项
    • 这是嵌套无序列表的一项
    • 这是嵌套无序列表的一项
    • 这是嵌套无序列表的一项

为了输入起来方便我通常用 -

待办事项（TODO List）​

语法在无序列表的前缀基础上修改即可：

未完成的：- + 一个空格 + [ + 一个空格 + ] + 一个空格 + 内容

已完成的：- + 一个空格 + [x] + 一个空格 + 内容

吾日三省吾身

- [ ] 实验做完了吗
- [ ] 论文投稿了吗
- [x] 上班摸鱼了吗

渲染结果如下，在 Typora 中，渲染后的项目是可以点击以切换完成状态的：

http://localhost:3000

吾日三省吾身

  • 实验做完了吗
  • 论文投稿了吗
  • 上班摸鱼了吗

引用区块​

引用区块语法是：> + 一个空格 + 内容

例如：

> 我曾经说过：
>
> > 鲁迅曾经说过：我没有说过
>
> 我曾经说过：出门要戴口罩

渲染结果如下：

http://localhost:3000

我曾经说过：

鲁迅曾经说过：我没有说过

我曾经说过：出门要戴口罩

链接和图片​

添加链接和图片的语法类似：

链接：[ + 链接文字 + ]( + url + )，图片只需要在最前面加上一个 ! 即可

另外，在 url 后面 + 一个空格 + 引号包裹的一段文字，可以设置鼠标悬浮的提示文本（图片标题）

例如：

[李旻的主页](https://limina.top/ "欢迎来看看")

![李旻的 Github 头像](https://avatars.githubusercontent.com/u/60998275?v=4 "图有点糊")

渲染结果如下：

http://localhost:3000

李旻的主页

表格​

使用 | 分隔单元格，- 分隔表头和其他行

:- 设置左对齐，-: 设置右对齐，:-: 设置居中

例如：

| 这是一个 | 没有格式的 | 表格 |
| -------- | ---------- | ---- |
| 内容     | 内容       | 内容 |

| 这是   | 设置对齐的 |   表格 |
| :----- | :--------: | -----: |
| 左对齐 |    居中    | 右对齐 |

渲染结果如下：

http://localhost:3000
这是一个没有格式的表格
内容内容内容
这是设置对齐的表格
左对齐居中右对齐

代码块​

行内代码可以用一对反引号 ` 包裹，而一个代码块可以用三对反引号包裹（需要换行）

另外，对于代码块可以在三个反引号后指定语言，方便软件的自动高亮

由于常用的高亮规则为 Prismjs，建议根据官网支持统一语言的写法

例如：

行内代码 `printf("Hello world!")`

一个代码块

```php
<?php
echo 'hello world!';
?>
```

渲染结果如下：

http://localhost:3000

行内代码 printf("Hello world!")

一个代码块

<?php
echo 'hello world!';
?>

转义​

用上述不同的符号包裹文字可以实现不同的格式，如果需要显示特定的符号则需要使用转义字符

Markdown 使用反斜杠 \ 转义特殊字符，例如：

**这段文本被加粗了**

\*\*这段文本两边有两对星号\*\*

渲染结果如下：

http://localhost:3000

这段文本被加粗了

**这段文本两边有两对星号**

支持 HTML​

Markdown 对 HTML 有一定的支持，可以直接在文档里使用 HTML 编写 Markdown 原本无法设置的样式，如：

使用 <kbd>Ctrl</kbd> + <kbd>Alt</kbd> + <kbd>Del</kbd> 重启电脑

<span style="color: pink; font-family: SimSun; font-size: 2rem; font-weight: bold;">这是粉色宋体 3 号加粗文字</span>

中间有   一个空格

中间有&nbsp;&nbsp;&nbsp;三个空格

这会被渲染为：

http://localhost:3000

使用 Ctrl + Alt + Del 重启电脑

这是粉色宋体 3 号加粗文字

中间有 一个空格

中间有   三个空格

Markdown 的导出​

Markdown 编写的文档可以导出 HTML 、Word、图像、PDF、Epub 等多种格式的文档，这在两个软件中都有对应的实现

  • 导出 Word 的一个优点在于不再需要手动去调整目录，但是特定样式还是需要另外调整
  • 在 Typora 中，应用了什么主题，导出的 PDF 就是什么样子

另一种导出 Markdown 的方式是 Pandoc（实际上其它软件的导出功能几乎也都是基于它的），我在另一篇文档中会讲到
