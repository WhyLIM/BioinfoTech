如何下载 TCGA 的数据

以结肠腺癌 Colon adenocarcinoma (COAD) 的 RNA-Seq 数据为例

使用 R 包 TCGAbiolinks 下载​

安装 TCGAbiolinks​

BiocManager::install("TCGAbiolinks")

检索需要下载的数据​

query <- GDCquery(project = "TCGA-COAD",
                  data.category = "Transcriptome Profiling",
                  data.type = "Gene Expression Quantification",
                  experimental.strategy = "RNA-Seq",
                  workflow.type = "STAR - Counts")

参数说明：

  • project = "TCGA-COAD"：指定要查询的TCGA项目，这里是结直肠腺癌（Colorectal adenocarcinoma）的缩写TCGA-COAD。
  • data.category = "Transcriptome Profiling"：指定要查询的数据类别，这里是转录组测序数据。
  • data.type = "Gene Expression Quantification"：指定要查询的具体数据类型，这里是基因表达量数据。
  • experimental.strategy = "RNA-Seq"：指定要查询的实验策略，这里是RNA测序（RNA-Seq）。
  • workflow.type = "STAR - Counts"：指定要查询的分析流程类型，这里是STAR对测序数据进行计数的流程。
我不知道这些参数名

使用 getGDCprojects()$project_id 得到各个癌种的项目id，总共有52个ID值

下载数据​

GDCdownload(query)

如不设置特定的存储文件夹，TCGAbiolins会在工作目录下新建一个名为GDCdata的文件夹用来存储下载的数据文件，数据有特定的命名和组织形式

准备数据​

data <- GDCprepare(query)

GDCprepare()函数用于准备从GDC（Genomic Data Commons）下载的TCGA数据，以便后续的分析和处理。

具体而言，GDCprepare()函数在数据下载后执行以下操作：

  1. 数据校验和完整性检查：函数会验证下载的数据文件是否完整且没有损坏。它还会检查是否存在与数据关联的其他文件，例如临床信息文件。
  2. 数据整理和组织：函数会将下载的数据文件整理成方便后续分析的格式，例如将基因表达矩阵和临床数据整合到一个数据框中。这样，数据就可以更方便地进行处理和分析。
  3. 数据标准化：函数会对数据进行标准化处理，以确保不同样本之间的比较具有可靠性和可重复性。这可能包括对基因表达数据进行批次校正、标准化或归一化。
  4. 基因和样本筛选：根据需要，函数可以进行基因和样本的筛选，选择感兴趣的基因或特定的样本子集进行后续分析。
  5. 添加附加信息：函数还可以添加额外的注释信息，如基因符号、癌症类型等，以帮助进一步分析和解释数据。

通过执行GDCprepare()函数，你可以对从GDC下载的TCGA数据进行预处理和整理，以便进行后续的数据分析、可视化和统计等操作。它提供了一个方便且一致的数据准备流程，以帮助简化数据处理过程并提供可靠的分析结果。

  • Starting to add information to samples：开始为样本添加信息。
  • => Add clinical information to samples：向样本添加临床信息。
  • => Adding TCGA molecular information from marker papers：从标记论文中添加TCGA分子信息。
  • => Information will have prefix 'paper_'：添加的信息将以'paper_'作为前缀。
  • coad subtype information from:doi:10.1038/nature11252：提供了肠直肠腺癌（COAD）亚型信息的引用来源。
  • Available assays in SummarizedExperiment：在SummarizedExperiment对象中可用的分析数据类型。

接下来输出列出了可用的分析数据类型（assays）：

  • unstranded：unstranded表示测序数据没有指定方向性信息。在这种情况下，无法区分来自于正链（sense strand）还是反链（antisense strand）的转录本。因此，无法确定基因的表达方向。
  • stranded_first：stranded_first表示测序数据具有方向性信息，其中只有来自正链的转录本进行了测序。这意味着只有来自正链的基因表达信息被记录和计量，反链的信息被忽略。
  • stranded_second：stranded_second表示测序数据具有方向性信息，其中只有来自反链的转录本进行了测序。与stranded_first相反，这里只记录和计量反链的基因表达信息。
  • tpm_unstrand：tpm_unstrand表示基因表达计量采用了TPM（Transcripts Per Million）的方法，并且没有考虑测序数据的方向性。TPM是一种常用的表达量计算方法，它校正了基因长度和测序深度的影响，并将基因表达量按百万分之一进行归一化。
  • fpkm_unstrand：fpkm_unstrand表示基因表达计量采用了FPKM（Fragments Per Kilobase of transcript per Million mapped reads）的方法，并且没有考虑测序数据的方向性。FPKM是另一种常用的表达量计算方法，它校正了基因长度和测序深度的影响，并将基因表达量按每千碱基每百万读数进行归一化。
  • fpkm_uq_unstrand：fpkm_uq_unstrand表示基因表达计量采用了FPKM-UQ（Upper Quartile normalized FPKM）的方法，并且没有考虑测序数据的方向性。FPKM-UQ是在FPKM的基础上进行了上四分位数归一化，可以更好地消除批次效应和基因长度的偏差。
