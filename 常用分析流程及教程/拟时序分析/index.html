单细胞数据（loom）的拟时序分析

此流程以 GSE156793 为例，这是一套人类胎儿组织的单细胞 RNA-seq 数据

所需文件：

  • GSE156793_S3_gene_count.loom.gz
  • GSE156793_S1_metadata_cells.txt.gz（可以不用）
  • GSE156793_S2_Metadata_genes.txt.gz（可以不用）
拟时序分析是什么？

拟时序分析（Pseudotime Analysis）是一种用于分析单细胞转录组数据的方法，旨在研究细胞发育、分化或其他动态过程中的时序性变化。

在单细胞转录组数据中，每个细胞的转录组信息可以看作是细胞状态的快照。拟时序分析通过对细胞之间的转录组差异进行建模和分析，试图推断细胞之间的时序关系和发育轨迹。

拟时序分析的关键是识别和定义一种“伪时间”（pseudotime）的概念，用于表示细胞发育或动态过程中的连续时间轴。伪时间可以被认为是一种细胞状态的排序，表征了细胞在发育过程中的相对顺序。

loom 文件的结构​

与标准的 R 对象将其中包含的所有数据加载到内存中不同，loom 对象只是连接到磁盘上的一个文件，这使得它能够以低内存读取大规模的数据集

> lfile
Class: loom
Filename: /home/mli/Study/DeTumor/GSE156793_S3_gene_count.loom
Access type: H5F_ACC_RDWR
Attributes: CreationDate, LOOM_SPEC_VERSION, last_modified, version
Listing:
       name    obj_type    dataset.dims dataset.type_class
  col_attrs   H5I_GROUP            <NA>               <NA>
 col_graphs   H5I_GROUP            <NA>               <NA>
     layers   H5I_GROUP            <NA>               <NA>
     matrix H5I_DATASET 4062980 x 63561          H5T_FLOAT
  row_attrs   H5I_GROUP            <NA>               <NA>
 row_graphs   H5I_GROUP            <NA>               <NA>

根据输出可以发现 loom 文件的结构可以分为以下几个部分：col_attrs、col_graphs、layers、matrix、row_attrs和row_graphs

  • col_attrs：该组存储与列（细胞）相关的属性信息。这些属性可以包括细胞类型、样本信息、细胞状态等。通常，每一列（细胞）都会有一组属性信息与之关联。
  • col_graphs：该组存储与列（细胞）相关的图形数据。例如，如果有关系网络或细胞相似性图等与细胞相关的图形数据，可以存储在这个组中。
  • layers：该组存储数据的层信息。在单细胞RNA测序中，通常会有不同的数据层，例如基因表达水平、转录本拼接信息等。每个数据层都可以存储在这个组中。
  • matrix：该组存储基因表达矩阵数据，这个基因表达矩阵是单细胞RNA测序数据的核心部分，可以用于后续的数据分析、细胞聚类、差异表达分析、细胞发育轨迹推断等任务。从结果可知它的维度为 4062980 行 x 63561 列，其数据类型为浮点型（H5T_FLOAT）。其中，该矩阵的列表示细胞，行表示基因。
  • row_attrs：该组存储与行（基因）相关的属性信息。这些属性可以包括基因名称、基因功能注释等。每一行（基因）都会有一组属性信息与之关联。
  • row_graphs：该组存储与行（基因）相关的图形数据。例如，如果有基因共表达网络或基因相似性图等与基因相关的图形数据，可以存储在这个组中。
tip

需要注意的是，不同的 loom 文件可能会使用不同的对象名存储细胞、基因名称等信息，因此官方教程中的访问方式并不一定适用于所有数据集，需要根据具体情况来确定正确的对象名

ls(lfile[["col_attrs"]])

lfile[["col_attrs"]] 表示 lfile 对象中的 col_attrs 组，col_attrs 组中包含了许多列属性，比如细胞名称、实验批次、样本组等信息。这些列属性在 loom 文件中使用 HDF5 格式存储。

这些属性包括：

  • All_reads：该细胞的总读数。
  • Assay：Assay 的名称。
  • Batch：表示该细胞所属的批次。
  • Development_day：样本采集的开发天数。
  • Exon_reads：该细胞的外显子读数。
  • Experiment_batch：表示该实验所属的批次。
  • Fetus_id：胎儿的 ID。
  • Intron_reads：该细胞的内含子读数。
  • Main_cluster_name：该细胞所属的主要簇名称。
  • Main_cluster_umap_1：该细胞在 UMAP 图中的 x 坐标。
  • Main_cluster_umap_2：该细胞在 UMAP 图中的 y 坐标。
  • Organ：该细胞所属的组织器官。
  • Organ_cell_lineage：该细胞所属的器官细胞系谱。
  • RT_group：该细胞所属的 RT 组。
  • Sex：该细胞所属个体的性别。
  • Size_Factor：归一化的大小因子。
  • batch：样品所属批次。
  • obs_names：该列对应的观测名称。
  • sample：样品的名称。

创建 Seurat 对象​

实际上，loom 文件也可以支持 Seurat 的基本函数，但常用的一种格式还是 Seurat 对象，所以先由 loom 创建 Seurat 对象

loom 文件中，row.attrs 表示基因，col.attrs 表示细胞，HDF5 库对底层数据矩阵的访问进行了转置，因此需要转置回来

runSeurat <- function(Organ){
    print(paste0(Organ, ' start'))
    # 1.连接到 loom 文件
    lfile <- connect("data/GSE156793_S3_gene_count.loom", mode = "r")
    
    # meta 信息文件如果没有提供，也可以从 loom 文件构建，但构建出来的可能会与提供的文件存在一些差异，有些分析无法进行
    # 直接读取：
    meta <- read.csv("data/GSE156793_S1_metadata_cells.txt", header = TRUE, row.names = 1, stringsAsFactors = F)
    # 从 loom 文件构建：
    # cellid <- lfile[["col_attrs/sample"]][]
    # meta <- as.data.frame(cellid)
    # cell_colnames <- c()
    # for (i in names(lfile[["col_attrs"]])) {
    #   cell_colnames <- c(cell_colnames, i)
    #   group <- paste0("col_attrs/", i)
    #   meta <- cbind(meta, lfile[[group]][])
    # }
    # rownames(meta) <- meta[, 1]
    # meta <- meta[, -1]
    # colnames(meta) <- cell_colnames
    
    # gene <- read.csv("GSE156793_S2_Metadata_genes.txt", header = TRUE)
    
    # 2.提取 matrix
    rowid <- which(lfile[["col_attrs/Organ"]][] == Organ)
    matrix <- lfile[["matrix"]][rowid[1]:rowid[length(rowid)], ]
    # dim(matrix)
    # matrix 的行为基因，列为细胞，需要转置才符合 Seurat 对象中的矩阵
    matrix <- t(matrix) 
    # dim(matrix)
    # 提取基因名和细胞名
    gene <- lfile[["row_attrs/gene_short_name"]][]
    cellid_sele <- lfile[["col_attrs/sample"]][][rowid[1]:rowid[length(rowid)]]
    # 把基因名和细胞名加入矩阵
    colnames(matrix) <- cellid_sele
    row.names(matrix) <- gene
    # 创建 seurat 对象
    seurat_data <- CreateSeuratObject(counts = matrix,
                                      project = Organ, # 标识，用来标识该数据集所属的项目
                                      min.cells = 3, # 指定了一个细胞必须具有的最低基因表达数量，细胞中基因的数量低于这个阈值的细胞将被过滤掉
                                      min.features = 200 # 指定了一个细胞必须具有的最低基因特征数，细胞中具有的非零表达的基因数量低于这个阈值的细胞将被过滤掉
                                     )
    # 提取组织的 metadata
    # cell_id <- colnames(seurat_data)
    # meta <- meta[cell_id, ]
    meta <- meta[cellid_sele, ]
    # 添加分群信息
    Idents(seurat_data) <- meta$Organ_cell_lineage
    # run umap
    seurat_data <- FindVariableFeatures(seurat_data, selection.method = "vst", nfeatures = 2000)
    seurat_data <- NormalizeData(seurat_data)
    all.genes <- rownames(seurat_data)
    seurat_data <- ScaleData(seurat_data, features = all.genes)
    seurat_data <- RunPCA(seurat_data, features = VariableFeatures(object = seurat_data))
    seurat_data <- RunUMAP(seurat_data, dims = 1:20)
    # 替换 umap 值
    seurat_data@reductions$umap@cell.embeddings[, 1] <- meta$Main_cluster_umap_1
    seurat_data@reductions$umap@cell.embeddings[, 2] <- meta$Main_cluster_umap_2

    p <- DimPlot(seurat_data, reduction = "umap")
    ggsave(paste0("result/DimPlot_umap_", Organ, ".pdf"), plot = p, width = 12, height = 5)
    saveRDS(seurat_data, file = paste0("result/", Organ, ".rds"))
    print(paste0(Organ, " end"))
}

runSeurat(Organ = 'Stomach')

以上的函数根据器官来创建 Seurat 对象（.rds 数据）

准备 monocle3 的输入数据​

使用 monocle3 的第一步是将数据加载到 monocle 3 的主类 cell_data_set 中：

cds <- new_cell_data_set(expression_matrix,
                         cell_metadata = cell_metadata,
                         gene_metadata = gene_annotation)

  • expression_matrix：表达值的数字矩阵，行是基因，列是 cell；

  • cell_metadata：一个数据框，行是cell，列是cell属性(如细胞类型，培养条件，天数等)；

  • gene_metadata：一个数据框，行是特征（基因），列是基因属性，如生物类型，gc内容等。

表达值矩阵必须：

  1. 拥有与cell_metadata的行数相同的列数；
  2. 拥有与gene_metadata的行数相同的行数。

另外：

  1. cell_metadata：对象的行名称应该与表达式矩阵的列名匹配；
  2. gene_metadata：对象的行名应该匹配表达式矩阵的行名；
  3. gene_metadata：一列应该命名为“gene_short_name”，它代表每个基因的基因符号或简单名称(通常用于绘图)。
