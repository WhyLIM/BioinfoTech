R&RStudio

PC 机的 R&RStudio 安装​

安装 R​

教程可以参考 南京大学镜像站

以下是一个简单的示例

添加仓库​

sudo gedit /etc/apt/sources.list.d/r-mirror.list

写入如下内容，其中 jammy 是 Ubuntu 22.04 的代号，需根据版本自行替换

deb https://mirror.nju.edu.cn/CRAN/bin/linux/ubuntu jammy-cran40/

添加公钥​

sudo apt-key adv --keyserver keyserver.ubuntu.com --recv-keys E298A3A825C0D65DFD57CBB651716619E084DAB9

在新版本的 Ubuntu 中，这是一项不太优雅的做法，会在 update 时出现警告：

W: https://mirror.nju.edu.cn/CRAN/bin/linux/ubuntu/jammy-cran40/InRelease: Key is stored in legacy trusted.gpg keyring (/etc/apt/trusted.gpg), see the DEPRECATION section in apt-key(8) for details.

这似乎是因为 apt 1.4 及更高版本默认使用 trusted.gpg.d 目录，而不再使用旧的 trusted.gpg

如果你像我一样是强迫症和懒癌患者，有一个快速但不算太优雅的方法可以解决这个问题：

sudo cp /etc/apt/trusted.gpg /etc/apt/trusted.gpg.d

安装 R​

sudo apt update
sudo apt install r-base-dev

常用 R 包的安装及注意事项​

依赖包的安装​

一些 R 包需要的编译工具系统可能是不自带的，功能类似于 Windows 下的 Rtools

在常见的两种 Linux 发行版中，这些依赖可以通过安装这些包解决：

  • Ubuntu
  • CentOS 7
sudo apt install libfontconfig1-dev libcurl3-dev libxml2-dev libssl-dev cmake libgit2-dev libharfbuzz-dev libfribidi-dev libtiff-dev libicu-dev
sudo yum install openssl-devel fontconfig-devel libxml2-devel harfbuzz-devel fribidi-devel freetype-devel libpng-devel libtiff-devel libjpeg-turbo-devel cairo-devel libXt-devel proj-devel proj-nad proj-epsg geos-devel udunits2-devel hdf5-devel cmake gdal-devel
caution

其中 hdf5-devel 、 cmake 和 gdal-devel 等包的版本可能过低，需要手动安装高版本

这对不熟悉的人来说是一个非常麻烦的事情，所以没什么事还是用 Ubuntu 吧

BiocManager 的安装和问题​

BiocManager 提供了一种方便的方式用于管理 Bioconductor 软件包的安装和更新。

截至目前（2023-04-25）的最新版本为 3.17，适用于 R 4.3.0

一般来说，未安装过 BiocManager 包的会默认安装适用于当前 R 语言版本的最新版本，但我们也可以手动更新版本：

if (!require("BiocManager", quietly = TRUE))
    install.packages("BiocManager")
BiocManager::install(version = "3.17")

安装包的默认位置是在类似 /usr/lib/R/library，而这个位置是需要 root 权限的

按照 Rstudio 的报错修改目录权限

sudo chmod 777 /usr/lib/R/library

经典的 clusterProfiler 包安装​

最近 KEGG 更换了 API，较新版本的 clusterProfiler 中才能进行 KEGG 富集分析，建议是安装 3.16 版本及以上的 BiocManager，其中包含了较新版本的 clusterProfiler

BiocManager::install("clusterProfiler")
# ggplot2 已经包含在 clusterProfiler 中
# BiocManager::install("ggplot2")

安装时通常会出现几个包无法更新的问题，通常是以下几个包：boot，foreign，lattice，MASS，Matrix，spatial，survival

解决办法也很简单：删掉重装就行了

sudo rm -rf /usr/lib/R/library/boot
sudo rm -rf /usr/lib/R/library/foreign
sudo rm -rf /usr/lib/R/library/lattice
sudo rm -rf /usr/lib/R/library/MASS
sudo rm -rf /usr/lib/R/library/Matrix
sudo rm -rf /usr/lib/R/library/spatial
sudo rm -rf /usr/lib/R/library/survival
BiocManager::install("boot")
BiocManager::install("foreign")
BiocManager::install("lattice")
BiocManager::install("MASS")
BiocManager::install("Matrix")
BiocManager::install("spatial")
BiocManager::install("survival")

其他常用包​

BiocManager::install("org.Hs.eg.db")
BiocManager::install("tidyverse")
BiocManager::install("ggpubr")
BiocManager::install("limma")
BiocManager::install("do")
BiocManager::install("devtools")

如何定位报错​

如果你使用的是 RStudio 的默认主题，那么报错信息通常位于输出的红色文字和黑色文字交界

tip

不要因为看着烦/看不懂就不去看报错

安装 R 包时常见的两类报错​

  • 编译错误
  • 更新错误

  1. 一种是直接告诉你的，例如：

    * installing *source* package ‘openssl’ ...
    ** package ‘openssl’ successfully unpacked and MD5 sums checked
    ** using staged installation
    Using PKG_CFLAGS=
    --------------------------- [ANTICONF] --------------------------------
    Configuration failed because openssl was not found. Try installing:
     * deb: libssl-dev (Debian, Ubuntu, etc)
     * rpm: openssl-devel (Fedora, CentOS, RHEL)
     * csw: libssl_dev (Solaris)
     * brew: openssl (Mac OSX)
    If openssl is already installed, check that 'pkg-config' is in your
    PATH and PKG_CONFIG_PATH contains a openssl.pc file. If pkg-config
    is unavailable you can set INCLUDE_DIR and LIB_DIR manually via:
    R CMD INSTALL --configure-vars='INCLUDE_DIR=... LIB_DIR=...'
    -------------------------- [ERROR MESSAGE] ---------------------------
    tools/version.c:1:30: fatal error: openssl/opensslv.h: No such file or directory
     #include <openssl/opensslv.h>
                                  ^
    compilation terminated.
    --------------------------------------------------------------------
    ERROR: configuration failed for package ‘openssl’

    根据提示去安装对应的包即可

  2. 一种是比较隐晦的，例如：

    *** *********************************************************************
    *** stringi cannot be built.
    *** Failed to download the ICU data library (icudt). Stopping now.
    *** For build environments that have no internet access,
    *** see the INSTALL file for a workaround.
    *** *********************************************************************
    ERROR: configuration failed for package ‘stringi’
    * removing ‘/home/limin/R/x86_64-pc-linux-gnu-library/4.1/stringi’
    Warning in install.packages :
      installation of package ‘stringi’ had non-zero exit status
    * installing *source* package ‘pkgbuild’ ...

    常有类似 cannot be built 或 configuration failed 的字样

    查看上下文，可以看到前文有类似这样的信息：

    checking whether we may compile src/icu69/common/putil.cpp... yes
    checking whether we may compile src/icu69/i18n/number_affixutils.cpp... yes
    checking whether alignof(std::max_align_t) is available... yes
    checking whether the ICU data library can be downloaded... downloading the ICU data library (icudt)
    output path: icu69/data/icu4c-69_1-data-bin-l.zip
    trying URL 'https://raw.githubusercontent.com/gagolews/stringi/master/src/icu69/data/icu4c-69_1-data-bin-l.zip'
    Error in download.file(paste(href, fname, sep = ""), icudtzipfname, mode = "wb"): cannot open URL 'https://raw.githubusercontent.com/gagolews/stringi/master/src/icu69/data/icu4c-69_1-data-bin-l.zip'
             
    trying URL 'http://raw.githubusercontent.com/gagolews/stringi/master/src/icu69/data/icu4c-69_1-data-bin-l.zip'
    Error in download.file(paste(href, fname, sep = ""), icudtzipfname, mode = "wb"): cannot open URL 'http://raw.githubusercontent.com/gagolews/stringi/master/src/icu69/data/icu4c-69_1-data-bin-l.zip'
             
    icudt download failed
    Error: Stopping on error

    checking whether... 的部分就是系统在查找编译工具，如果找不到，最后通常是 no

    有时系统会自己去下载缺失的编译工具，但更常见的是没有自动下载/下载不成功

    在上文中可以看到缺少 ICU 这个编译库，打开终端搜索此库

    sudo apt update
    sudo apt-cache search icu

    输出有时候会很多，可以细化一下

    sudo apt-cache search icu | grep lib | grep icu

    通常会找到一个名为（或类似） libicu-dev 的库

    libicu-dev - Development files for International Components for Unicode

    下载这个库就能解决问题

    sudo apt install libicu-dev
    caution

    但是在 CentOS 7 上这个问题也许会非常复杂，原因是 yum 软件源中的 icu 库版本太低（50.2）

    解决方法是：

    1. 从报错中的网址 https://raw.githubusercontent.com/gagolews/stringi/master/src/icu69/data/icu4c-69_1-data-bin-l.zip 下载压缩包，这是 icu 69.1 版本的数据文件，包含了所需的二进制数据文件和配置信息

      wget https://raw.githubusercontent.com/gagolews/stringi/master/src/icu69/data/icu4c-69_1-data-bin-l.zip

    2. 下载 stringi 的压缩包

      wget https://mirrors.e-ducation.cn/CRAN/src/contrib/stringi_1.7.12.tar.gz

    3. 手动安装

      手动安装时需要在终端中使用 --configure-vars 参数指定编译时所用的 icu 库位置，例如我将上述两个文件都下载在 /home/mli/Download/ 目录，因此命令这样写：

      R CMD INSTALL --configure-vars='ICUDT_DIR=/home/mli/Download/' stringi_1.7.12.tar.gz

有时安装新的 R 包时会询问要不要更新老包，有时不更新不影响当前 R 包的使用，但作为强迫症患者我一般选择全部更新

有些包由于目录权限或是一些未知原因无法更新，会有类似如下报错

* installing *source* package ‘MASS’ ...
** package ‘MASS’ successfully unpacked and MD5 sums checked
mv: cannot move '/usr/lib/R/library/MASS' to '/usr/lib/R/library/00LOCK-MASS/MASS': Permission denied
ERROR: cannot remove earlier installation, is it in use?
* removing ‘/usr/lib/R/library/MASS’

解决办法是到这些包所在目录（通常是 /usr/lib/R/library，具体看报错），将其删除，然后重新安装这些包

服务器的 R&RStudio 安装（CentOS）​

安装 R​

可以参考 安装 R 的官方文档

以下是一个简单的示例

添加 Extra Packages for Enterprise Linux (EPEL) 存储库

sudo yum install https://dl.fedoraproject.org/pub/epel/epel-release-latest-7.noarch.rpm

指定 R 版本

export R_VERSION=4.3.0

下载并安装 R

curl -O https://cdn.rstudio.com/r/centos-7/pkgs/R-${R_VERSION}-1-1.x86_64.rpm
sudo yum install R-${R_VERSION}-1-1.x86_64.rpm

验证 R 安装

/opt/R/${R_VERSION}/bin/R --version

如果要在终端中直接使用 R，添加软链接到 /usr/bin

sudo ln -s /opt/R/${R_VERSION}/bin/R /usr/bin/R
sudo ln -s /opt/R/${R_VERSION}/bin/Rscript /usr/bin/Rscript

安装 RStudio​

可以参考 安装 RStudio 的官方文档

以下是一个简单的示例

wget https://download2.rstudio.org/server/centos7/x86_64/rstudio-server-rhel-2023.03.0-386-x86_64.rpm
sudo yum install rstudio-server-rhel-2023.03.0-386-x86_64.rpm

安装后会自动打开 RStudio 服务，有些 CentOS 可能没有 service 命令，需要安装

sudo yum install initscripts -y

查看 RStudio 服务状态

sudo service rstudio-server status
